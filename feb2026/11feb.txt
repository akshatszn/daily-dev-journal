 Dev Journal — Promises & Async/Await

 Topic

JavaScript Promises and Async/Await

---

 Concept

JavaScript runs many operations **asynchronously**, meaning some tasks take time (like API calls, file loading, timers). To handle these tasks without blocking the program, JavaScript uses **Promises**.

A **Promise** represents a value that will be available **now, later, or never**.

A Promise has three states:

* **Pending** → the operation is still running
* **Fulfilled** → the operation completed successfully
* **Rejected** → the operation failed

Example:

```javascript
const promise = new Promise((resolve, reject) => {
  let success = true;

  if (success) {
    resolve("Task completed");
  } else {
    reject("Task failed");
  }
});
```

To get the result of a promise we use `.then()` and `.catch()`.

```javascript
promise
  .then(result => console.log(result))
  .catch(error => console.log(error));
```

---

 Async / Await

**Async/Await** is a cleaner way to work with promises.

* `async` makes a function return a promise.
* `await` pauses the execution of the function until the promise resolves.

Example:

```javascript
async function getData() {
  const response = await fetch("https://api.example.com/data");
  const data = await response.json();
  console.log(data);
}
```

This code behaves asynchronously but **looks like synchronous code**, which makes it easier to read and maintain.

---

 Difference Between `.then()` and `async/await`

| Promise (.then)                          | Async/Await                       |
| ---------------------------------------- | --------------------------------- |
| Uses chaining with `.then()`             | Uses sequential style             |
| Can become messy with many chained calls | Cleaner and easier to read        |
| Error handling with `.catch()`           | Error handling with `try...catch` |
| Older pattern                            | Modern and preferred approach     |

Example comparison:

Using `.then()`:

```javascript
fetch(url)
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => console.log(err));
```

Using `async/await`:

```javascript
async function getData() {
  try {
    const res = await fetch(url);
    const data = await res.json();
    console.log(data);
  } catch (err) {
    console.log(err);
  }
}
```

---

Confusion I Had

At first I thought **async/await replaces promises**, but actually **async/await is just a cleaner way to handle promises**. Promises still exist behind the scenes.

---

 Key Takeaways

* JavaScript handles asynchronous tasks using **Promises**.
* Promises represent future results of operations.
* `.then()` is used to handle resolved promises.
* `async/await` provides a cleaner syntax for working with promises.
* `await` can only be used inside an `async` function.
